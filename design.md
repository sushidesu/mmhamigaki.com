# mmhamigaki.com

- mmhgamigaki.com は mmhamigaki のあらゆるコンテンツを閲覧できる コンテンツプラットフォームである
- シンプルさと閲覧性を重視し、 UIのベースは YouTube を参考にする
- primary colorは #febec2 とする

## 詳細

### トップページ 及びその関連ページ

以下の要素で構成される, YouTubeのチャンネルトップページと近い形式

- ヘッダー
- mmhamigakiの概要
- 投稿一覧
  - "重要": 異なるアスペクト比のサムネイルがタイル状に並ぶ配置にする (Eagleという画像ぎゃらりーアプリ参考)
  - 一覧にはコンテンツのサムネイル, コンテンツのタイトルを表示する
  - コンテンツには以下の種類があるが, 一律に扱う, ただしサムネイルのアスペクト比などでどの種類のコンテンツなのかはわかる (なので特別どの種類のコンテンツであるかは表示しなくて良い)
    - ポスト
      - ブログのようなもの
      - アスペクト比は基本的に 16:9
    - 音楽
      - 音楽を流せる, SoundCloudのように
      - アスペクト比は基本的に 1:1
    - イラスト
      - イラスト
      - 4コマ漫画, 複数ページ漫画も投稿したい
      - アスペクト比はイラストに準拠 (任意)
    - LIVE
      - アスペクト比は 16:9
      - LIVE配信ができる YouTubeのように
    - 動画
      - アスペクト比は 16:9
      - 動画を流せる, YouTubeのように
- コンテンツ詳細
  - コンテンツはすべてユニークなslugを持ち、詳細ページでアクセスできる
    - 詳細ページはすべて /contents/{slug} とする
  - 基本的にベースは同じUIだが、一部コンテンツの種類に応じたUIが表示される

### 管理ページ

- コンテンツを管理するためのページ
- 認証はまだ作成していない
- コンテンツをリストするページと、詳細で編集するページがある

## 相談

- 様々な種類のコンテンツを一律に扱いたい, しかし個別のコンテンツに合わせた最適なUIを提供したい
- コンテンツのタイプを固定してしまうと, あとからのコンテンツタイプの変更や, 混合コンテンツを扱うことが難しくなってしまう
- これをどう解消すべきか考えたい (例えば, コンテンツの子要素として副数のタイプのアタッチメントをつけられる設計にする, など)
